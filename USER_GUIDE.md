## 1. 系统简介

### 1.1 功能概述

本系统是基于机器学习的**信贷风险评估工具**，可以：

| 功能 | 说明 |
|------|------|
| ✅ 单客户评估 | 输入客户信息，实时计算违约概率和风险等级 |
| ✅ 批量预测 | 上传 CSV 文件，批量处理多个客户 |
| ✅ 决策解释 | SHAP 分析，显示哪些因素影响风险 |
| ✅ 模型透明度 | 查看模型性能和特征重要性 |
| ✅ 历史记录 | 自动保存所有预测记录 |

### 1.2 技术背景

- **算法**: LightGBM 梯度提升
- **训练数据**: 108,657 条真实信贷记录
- **准确率**: AUC = 0.76+ (Out-of-Fold)
- **支持语言**: 中文 / English

**计算公式**:
- 月供收入比 = 月供 / 年收入
- 信贷收入比 = 贷款 / 年收入

### 3.3 开始评估

#### 3.3.1 提交评估

填写完所有信息后，点击 **🔍 开始评估** 按钮。

> **[截图占位符 8: 评估结果界面]**
> 说明：展示完整的评估结果，包括违约概率、风险等级、决策建议

#### 3.3.2 结果解读

**顶部指标卡片**:
- **违约概率**: 客户违约的可能性（百分比）
- **风险等级**: 低风险 ✅ / 中等风险 ⚠️ / 高风险 ❌
- **决策建议**: 建议批准 / 需要考量 / 建议拒绝

> **[截图占位符 9: 风险等级仪表盘]**
> 说明：展示彩色的风险等级仪表盘（绿色=低风险，黄色=中等风险，红色=高风险）

**风险等级划分**:

| 违约概率 | 风险等级 | 决策建议 | 颜色 |
|----------|----------|----------|------|
| < 阈值×33% | 低风险 | 建议批准 | 🟢 绿色 |
| 阈值×33% ~ 阈值 | 中等风险 | 需要考量 | 🟡 黄色 |
| ≥ 阈值 | 高风险 | 建议拒绝 | 🔴 红色 |

**默认阈值**: 0.24 (24%)

#### 3.3.3 决策依据分析

> **[截图占位符 10: SHAP 决策依据分析]**
> 说明：展示"有利因素"和"风险因素"列表

**有利因素**（绿色 ✅）: 降低违约风险的因素
- 例如：高收入、稳定工作、有房产等

**风险因素**（红色 ⚠️）: 增加违约风险的因素
- 例如：收入低、月供高、征信分低等

**SHAP 值解读**:
- 负值 = 降低风险（有利）
- 正值 = 增加风险（不利）
- 绝对值越大 = 影响越大

## 8. 常见问题

### Q1: 系统预测准确吗？

**A**: 模型 AUC 约为 0.76，意味着：
- 对76%的客户，模型能正确区分违约/不违约
- 但仍有24%的不确定性
- **建议**: 将预测结果作为参考，结合人工判断

### Q2: 为什么外部征信评分最重要？

**A**: EXT_SOURCE 是外部征信机构（如央行征信中心）提供的综合评分，综合了：
- 历史还款记录
- 负债情况
- 信用历史长度
- 查询次数

这些是预测违约最有效的信息。

### Q3: 如何提高自己的批准率？

根据特征重要性，建议：
1. **提高征信分**: 按时还款，降低负债
2. **增加收入证明**: 提供完整收入证明
3. **降低月供**: 延长贷款期限
4. **提供资产证明**: 房产、车产等

### Q4: 批量预测支持多少条数据？

**A**: 建议不超过 10,000 条，超过可能：
- 处理时间较长
- 内存占用较大
- 浏览器可能超时

### Q5: 预测历史保存在哪里？

**A**: 保存在 `predictions_history/history.json` 文件中，位于应用目录下。

### Q6: 如何备份数据？

**A**:
1. 定期导出预测历史（复制 history.json）
2. 下载批量预测结果（CSV格式）
3. 备份整个应用文件夹

### Q7: 系统需要联网吗？

**A**:
- 本地运行：不需要联网
- Streamlit Cloud 部署：需要联网访问

### Q8: 为什么有些字段是负数？

**A**: 如 DAYS_BIRTH（出生天数）使用负数表示：
- DAYS_BIRTH = -10000 → 约27岁
- 这是数据集的编码方式，系统会自动处理
